**Bug-Finder Spec**
Objective: make bug-finder outputs measurable and fair in a multi-agent competition loop for this repo, with mandatory impact classes `low/some/critical` and scores `+1/+5/+10`.

**Ground Truth**
1. `documentation/tests.md`: canonical expected behavior and release acceptance criteria (especially Section 8).
2. `tests/harness/scenarios/cases.yaml`: suite/case definitions (`A-*` to `H-*`).
3. `tests/harness/assertions.py`: invariant definitions (`all_docs_ready`, `chat_sources_scoped`, `each_doc_embeddings_match_chunks`, etc.).
4. `tests/harness/runner.py`: where runtime evidence is emitted.
5. `backend/app.py`: API contracts for `/api/health`, `/api/documents*`, `/api/chat`.
6. `backend/ingestion/pipeline.py`: ingestion state machine and persistence phases.

**Finding Unit**
A scored finding is one unique broken invariant with one primary root-cause signature:

`finding = (invariant_id, affected_surface, root_cause_signature)`

Examples of `affected_surface`: `api:/api/chat`, `suite:E-006`, `ingestion:pdf`, `storage:embeddings`.

**Impact Rubric**
| Impact | Score | Assign when | Repo-specific triggers |
|---|---:|---|---|
| `critical` | `10` | Security, data integrity, availability, or release-blocking break | Scope/citation leakage, canonical ingestion failure, cleanup integrity failure, API crash on valid inputs, medium-scale service failure |
| `some` | `5` | Deterministic functional regression with meaningful but contained impact | One extractor path broken, routing/retrieval wrong for a subset, stable misbehavior without total outage |
| `low` | `1` | Non-blocking defect with low blast radius and workaround | Minor metadata/progress inconsistencies, low-risk edge-case quality issues |

Severity assignment order:
1. If any Section 8 release acceptance criterion in `documentation/tests.md` fails, classify at least `critical`.
2. If scoped chat constraints fail (`chat_sources_scoped` behavior from `tests/harness/assertions.py`), classify `critical`.
3. If delete/delete-all integrity fails (`E-006`, `E-007` semantics), classify `critical`.
4. If deterministic but limited to one modality/file family without security/data-loss risk, classify `some`.
5. Otherwise classify `low`.

**Evidence Rules**
A finding is scorable only with all of these:
1. Reproduction proof: case ID(s), dataset selector, and exact request/probe context.
2. Observed proof: artifact path(s) and key output excerpt.
3. Expected proof: explicit expected behavior source from `documentation/tests.md` or `tests/harness/assertions.py`.
4. Localization proof: at least one relevant `backend/...` code reference.
5. Impact proof: blast radius statement (single file, type family, canonical suite, or scale profile).

Preferred artifact sources:
1. `harness_runs/<run_id>/snapshots/cases/<case_id>.json`
2. `harness_runs/<run_id>/events/timeline.jsonl`
3. `harness_runs/<run_id>/api/health.json`
4. `harness_runs/<run_id>/logs/*`

**Confidence Model**
Score each component in `[0,1]`:
1. `repro`: `1.0` repeated or deterministic; `0.7` one clean repro; `0.4` flaky/ambiguous.
2. `contract`: `1.0` explicit invariant violation; `0.6` inferred expectation; `0.0` missing contract.
3. `artifact`: `1.0` payload+snapshot+error/log; `0.7` two artifacts; `0.4` one artifact.
4. `localization`: `1.0` function-level refs; `0.6` file-level refs; `0.0` none.

Formula:
`confidence = 0.35*repro + 0.25*contract + 0.25*artifact + 0.15*localization`

Gates:
1. `critical`: `confidence >= 0.86` and `contract = 1.0`.
2. `some`: `confidence >= 0.72`.
3. `low`: `confidence >= 0.60`.
4. Below gate: `needs_validation`, score `0`, excluded from leaderboard.

**Dedup Rules**
1. Build fingerprint: `sha1(normalized(invariant_id|surface|symptom_signature|primary_code_ref))`.
2. Exact duplicate if fingerprint matches.
3. Near-duplicate if invariant and primary code ref match and symptom similarity >= `0.80`.
4. Merge duplicates into one canonical finding with merged `case_ids` and `occurrence_count`.
5. Keep highest confidence and highest justified impact on merged finding.
6. Duplicate findings add no additional points.

**Hallucination Gates**
All must pass or finding is unscorable:
1. `artifact_exists`: every cited artifact path exists.
2. `code_ref_exists`: every cited code reference exists.
3. `expected_contract_cited`: expected behavior source is explicit.
4. `observed_mismatch_proven`: observed result directly contradicts expected.
5. `counterexample_checked`: agent looked for existing handling before claiming missing logic.
6. `env_sanity_checked`: not solely an expected environment limitation.
7. `no_speculative_root_cause`: root cause is evidence-backed, or clearly labeled as hypothesis.
