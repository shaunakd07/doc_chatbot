[2026-03-03T19:52:53] codex

Below is a risk-aware **disprove-bug agent** policy that is aggressive on weak/low-risk reports, but conservative on high-risk findings so it does not over-challenge them.

Grounding used from repo patterns: `tests/harness/evidence.py`, `tests/harness/runner.py`, `tests/harness/assertions.py`, `tests/harness/api_client.py`, `tests/harness/db_inspector.py`, and confidence normalization style in `backend/services/router_service.py`.

**Design Goal**
Maximize expected score from challenges while limiting catastrophic false challenges on high-severity bugs.

For a bug with score `S` and disprove success probability `p_false`:

`EV(challenge) = S * p_false - 2S * (1 - p_false) = S * (3*p_false - 2)`

Risk-neutral challenge threshold is `p_false > 2/3`.

To control risk, the agent uses:
- Lower-confidence bound (`p_lcb`) instead of raw `p_false`.
- Severity-specific thresholds.
- Mandatory counter-evidence gates.
- Portfolio risk budget.

**Severity Policy**
Map score to severity:
- `low`: `S <= 3`
- `medium`: `4 <= S <= 6`
- `high`: `7 <= S <= 8`
- `critical`: `S >= 9`

| Severity | Max wrong prob | Min `p_lcb` to challenge | Abstain band | Required strong counter-evidence |
|---|---:|---:|---:|---|
| low | 0.30 | 0.70 | +/- 0.05 | 1 item, any group |
| medium | 0.20 | 0.80 | +/- 0.06 | 2 items, >=2 groups |
| high | 0.10 | 0.90 | +/- 0.07 | 2 items, must include `runtime` + `static` |
| critical | 0.04 | 0.96 | +/- 0.08 | 3 items, must include `runtime` + `static` + `state/logs` |

This makes high/critical challenges rare and evidence-heavy by design.

**Required Counter-Evidence Rules**
Admissible counter-evidence must be:
- Reproducible: explicit artifact and environment fingerprint.
- Causal: explains why exploit path is impossible or claim is invalid.
- Independent: from different groups (`runtime`, `static`, `state/logs`, `spec`).
- Auditable: linked to concrete outputs (test run, DB snapshot, API trace).

Disallowed as sole basis:
- Pure LLM argumentation.
- Single flaky non-repro.
- Missing logs interpreted as proof.

**Confidence and Risk Computation**
- Build prior from severity + reporter confidence.
- Update with evidence likelihood ratios (Bayesian/odds update).
- Estimate uncertainty `sigma` from disagreement across checks/models/reruns.
- Use conservative confidence: `p_lcb = clamp(p_false_hat - z_sev * sigma, 0, 1)`.
- Compute `ev_lcb = S * (3*p_lcb - 2)`.
- Compute downside risk: `risk_cost = 2S * (1 - p_lcb)`.

**Decision Policy**
- `challenge_disprove` if all are true:
  - Evidence gate passes for severity.
  - `p_lcb >= threshold[severity] + abstain_band[severity]`.
  - `ev_lcb > 0`.
  - No unresolved strong pro-bug evidence.
  - Portfolio budget allows `risk_cost`.
- `abstain` if:
  - Evidence gate fails.
  - `p_lcb` is in abstain band around threshold.
  - Uncertainty too high.
  - Missing critical telemetry or repro setup.
- `uphold_or_escalate` if:
  - `p_lcb < threshold[severity] - abstain_band[severity]`.
  - Or strong contradictory evidence supports bug validity.

**Portfolio Control (Batch Optimization)**
For multiple reports, maximize total `sum(ev_lcb)` under risk budget:
- Constraint: `sum(risk_cost) <= B`.
- Additional cap: high+critical challenges consume at most a small fraction of `B` (for example 25%).
- Select by highest `ev_lcb` first among eligible candidates.

**Pseudo-Algorithm**
```python
def decide_disprove(report, evidence_items, budget_remaining):
    S = report.score
    sev = map_severity(S)
    cfg = POLICY[sev]

    admissible = [e for e in evidence_items if is_admissible(e)]
    strong = [e for e in admissible if e.reliability >= 0.8 and e.reproducible]

    if not evidence_gate_passes(strong, cfg):
        return verdict("abstain", "insufficient_counter_evidence")

    p_prior = prior_false_prob(report.confidence, sev)
    p_hat, sigma = update_p_false_with_evidence(p_prior, strong)

    p_lcb = clamp(p_hat - cfg.z_value * sigma, 0.0, 1.0)
    ev_lcb = S * (3 * p_lcb - 2)
    risk_cost = 2 * S * (1 - p_lcb)

    if unresolved_pro_bug_evidence(report):
        return verdict("abstain", "conflicting_strong_evidence")

    if abs(p_lcb - cfg.threshold) <= cfg.abstain_band:
        return verdict("abstain", "confidence_gray_zone")

    if p_lcb < cfg.threshold:
        return verdict("uphold_or_escalate", "likely_valid_bug")

    if ev_lcb <= 0 or risk_cost > min(cfg.per_report_risk_cap, budget_remaining):
        return verdict("abstain", "risk_adjusted_ev_not_positive")

    return verdict("challenge_disprove", "high_confidence_counter_evidence")
```

This policy gives you adversarial pressure where it is safe, while strongly biasing toward abstain/escalate for high-risk findings unless counter-evidence is unusually strong.
