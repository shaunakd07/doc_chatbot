**Referee Protocol Design**

Reviewed existing harness outputs (`documentation/test_harness.md`) and artifact layouts from `tests/harness/runner.py` and `tests/harness/evidence.py`. Here is a deterministic referee adjudication and scoring design integrating with test harness artifacts.

1. **Protocol Steps**  
1. Round opens; bug finder submits claims (`T0-T1`).  
2. Disprover submits counterclaims (`T1-T2`).  
3. Evidence freeze at `T2`; claims locked.  
4. Referee adjudicates claims deterministically in lex order by `(case_id, claim_id)`.  
5. If unresolved or conflicting, trigger deterministic replay (3 runs) for tie-breaking.  
6. Oracle outcome finalized from replay majority.  
7. Scoring ledger updated; calibration metrics computed.  

2. **Evidence Standards**  
- **Tier A:** Structured, direct harness artifacts (e.g., `snapshots/cases/<case_id>.json` checks) — weight 3.  
- **Tier B:** Derived run-level metrics (e.g., `summary.json`, `api/health.json`) — weight 2.  
- **Tier C:** Unstructured logs (e.g., `logs/managed_app.log`) — weight 1.  
- Claims require ≥1 Tier A evidence pointer; otherwise rejected.

3. **Decision Matrix**

| Bug Claim Status     | Disprove Claim Status | Replay Result       | Verdict           | Winner      |  
|---------------------|----------------------|--------------------|-------------------|-------------|  
| Supported           | Unsupported          | —                  | BUG_VALID         | Bug finder  |  
| Unsupported         | Supported            | —                  | BUG_INVALID       | Disprover   |  
| Supported           | Supported            | —                  | Tie-break chain   | See tie-breakers |  
| Unsupported         | Unsupported          | —                  | Tie-break chain   | See tie-breakers |  
| Any Undecidable     | —                    | —                  | Tie-break chain   | See tie-breakers |  
| —                   | —                    | Replay Major. Bug valid | BUG_VALID       | Bug finder  |  
| —                   | —                    | Replay Major. Bug invalid | BUG_INVALID  | Disprover   |  
| —                   | —                    | Replay Inconclusive | INSUFFICIENT_EVIDENCE | Draw    |  

4. **Tie-Breakers (Deterministic Order)**  
1. Higher evidence tier total support wins.  
2. Count of independent facts at winning tier.  
3. Replay majority verdict (3 runs).  
4. Fallback verdict: `INSUFFICIENT_EVIDENCE` (draw).  

5. **Scoring Formulas**  

```python
referee_points_i = +1 if referee_verdict_i == oracle_verdict_i else -1
referee_total_score = sum(referee_points_i for i in adjudications)
accuracy = (referee_total_score + N) / (2 * N)  # N = settled adjudications
```

- Tri-state scoring applies for labels: `BUG_VALID`, `BUG_INVALID`, `INSUFFICIENT`.  
- No zero except while pending oracle result.

6. **Data Model Overview**

| Entity             | Key Fields                                   | Notes                             |

  - `harness_runs/<run_id>/tournament/scores/calibration.json` (calibration snapshot)  
  - `harness_runs/<run_id>/tournament/events/referee_timeline.jsonl` (referee events)  

8. **Calibration Metrics**

- Net score = sum of +1/-1 referee points.  
- Accuracy = fraction correct adjudications.  
- Expected Calibration Error (ECE) by confidence bins.  
- Brier score for multi-class decisions.  
- Overturn rate: fraction of decisions changed after replay.  
- Flaky rate: proportion of inconclusive oracle outcomes.  
- Determinism rate: fraction of adjudications reproducible byte-for-byte on rerun (target 1.0).  

**Recommended thresholds:**  
- Determinism rate ≥ 100%  
- Overturn rate < 10%  
- ECE < 0.08  
- Rolling accuracy > 0.6

**Referee Protocol**
Using the current harness outputs documented in `documentation/test_harness.md` and produced by `tests/harness/runner.py` + `tests/harness/evidence.py`, this is a deterministic design.

1. `Round Setup`
- Define `tournament_id`, `round_id`, submission deadline, replay policy (`N=3` recommended), and referee rule version.
- Freeze accepted artifact roots to exactly one run: `harness_runs/<run_id>/...`.

2. `Claim Intake`
- Bug-finder submits one structured claim per `case_id`.
- Disprove agent submits one counterclaim per same `case_id`.
- Claims without machine-checkable evidence pointers are rejected at intake.

3. `Evidence Freeze`
- Referee resolves artifact references and computes a canonical `evidence_bundle_hash`.
- Canonicalize JSON with sorted keys and normalized numbers/booleans.
- Store resolved values from pointers so later replay does not depend on parser differences.

4. `Deterministic Adjudication`
- Evaluate bug claim and disprove claim as `SUPPORTED`, `UNSUPPORTED`, or `UNDECIDABLE`.
- Use fixed decision matrix and fixed tie-breaker order.
- Emit `verdict`, `winner`, `confidence`, `rule_trace`, `tie_breaker_used`.

5. `Oracle Settlement For Referee Scoring`
- Independent settlement job computes oracle label from deterministic replay policy.
- Compare referee verdict vs oracle and award `+1` or `-1`.
- No self-grading: the same process that made the initial verdict should not settle itself.

6. `Audit + Replay`
- Every step writes append-only JSONL events plus hash-chained ledger records.
- Replay requires only run artifacts + claim files + referee rule version.

**Evidence Standards**
- `Tier A (Primary)`: `snapshots/cases/<case_id>.json` facts (checks, statuses, doc stats, chat results).
- `Tier B (Secondary)`: `summary.json`, `snapshots/run_context.json`, `events/timeline.jsonl`, `api/health.json`.
- `Tier C (Tertiary)`: `logs/managed_app.log` and other free text.
- Minimum acceptance rule: every claim must include at least one Tier A citation.
- Logs can explain cause but cannot override contradictory Tier A facts.

**Decision Matrix**
Use per-case claim outcomes after evidence validation:

| Bug Claim | Disprove Claim | Verdict | Winner |
|---|---|---|---|
| SUPPORTED | UNSUPPORTED | `BUG_VALID` | `bug_finder` |
| UNSUPPORTED | SUPPORTED | `BUG_INVALID` | `disprover` |
| SUPPORTED | SUPPORTED | tie-breakers | tie-break result |
| UNSUPPORTED | UNSUPPORTED | tie-breakers | tie-break result |
| Any UNDECIDABLE | any | tie-breakers | tie-break result |

Tie-breaker order (strict, deterministic):
1. Higher evidence tier wins (`A > B > C`).
2. More independent supporting facts at highest tier wins.
3. Replay majority on fixed `N=3` runs wins.
4. If still unresolved, output `INSUFFICIENT_EVIDENCE` and mark round as `draw`.

points_i = +1 if referee_verdict_i == oracle_verdict_i else -1

Recommended labels for oracle/verdict: `BUG_VALID`, `BUG_INVALID`, `INSUFFICIENT_EVIDENCE`.  
Exact-label match determines correctness.
